Implements a production-ready MVP for crypto strategy backtesting per the stakeholder scope.

Highlights
- FastAPI backend with multi-user JWT auth (access + refresh), PostgreSQL via SQLAlchemy 2.0, Alembic migrations
- Asset search & resolver via CoinGecko + Binance markets (CCXT)
- Hourly OHLCV ingestion from Binance with pagination, rate limit/backoff, and DB upsert caching
- RSI threshold strategy runner using vectorbt + pandas-ta (period/lower/upper) with fees/slippage
- RQ jobs (Redis) for backtests; worker processes ingest, run, and persist results
- Results persisted with aggregate + per-asset metrics, plus CSV outputs (equity, trades) saved under /data/backtests/<id>/
- React (Vite+TS) frontend with auth, new backtest wizard, runs list, result view (equity + drawdown charts, per-asset table, CSV download)
- Dockerized deployment: Postgres, Redis, backend API, RQ worker, and frontend (nginx) with healthchecks

Key endpoints
- Auth: POST /auth/register, /auth/login, /auth/refresh, GET /auth/me
- Assets: GET /api/v1/assets/search?q=btc, POST /api/v1/assets/resolve { cg_id }, GET /api/v1/assets
- Backtests: POST /api/v1/backtests, GET /api/v1/backtests, GET /api/v1/backtests/{id}, GET /api/v1/backtests/{id}/download?kind=equity|trades

Notes
- UTC timestamps (unix ms) throughout, unique(asset_id,timeframe,ts) enforced in price table
- Default timeframe 1h, lookback up to 3y via relative inputs (-3y/-180d/-72h)
- vectorbt with pandas-ta RSI (no TA-Lib dependency); aggregate equity is equal-weighted across assets
- Alembic ready with initial migration; backend container runs migrations on startup
- Frontend uses Axios with JWT/refresh interceptors and protected routes

Config (Docker defaults)
- DATABASE_URL=postgresql+psycopg2://postgres:postgres@postgres:5432/postgres
- REDIS_URL=redis://redis:6379/0
- JWT_SECRET=supersecret (change in prod), JWT_EXPIRE_MIN=30, JWT_REFRESH_EXPIRE_DAYS=14
- CORS_ORIGINS=http://localhost:5173,http://localhost:3000,http://frontend
- BINANCE_MARKET=spot, CCXT_RATE_LIMIT_MS=500

How to run
- cd docker && docker compose up --build
- Visit http://localhost:5173 for frontend, backend at http://localhost:8000 (GET /health)
