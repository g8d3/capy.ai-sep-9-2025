Implements a production-ready MVP for crypto strategy backtesting per the stakeholder scope.

Highlights
- FastAPI backend with multi-user JWT auth (access + refresh), PostgreSQL via SQLAlchemy 2.0, Alembic migrations
- Asset search & resolver via CoinGecko + Binance markets (CCXT)
- Hourly OHLCV ingestion from Binance with pagination, rate limit/backoff, and DB upsert caching
- RSI threshold strategy runner using vectorbt; RSI implemented via EWM for portability; equal-weighted aggregate
- RQ jobs (Redis) for backtests; worker processes ingest, run, and persist results
- Results persisted with aggregate + per-asset metrics, plus CSV outputs (equity, trades)
- React (Vite+TS) frontend with auth, new backtest wizard, runs list, result view (equity, drawdown, per-asset metrics, CSV download)
- Dockerized deployment (compose) + new scripts for non-docker deployment

Local (non-docker) scripts
- scripts/setup_local.sh — venv + pip install + frontend install/build (VITE_API_BASE_URL adjustable)
- scripts/start_local.sh — starts API on :8010, worker, and static frontend on :8080
- scripts/stop_local.sh — stops backgrounded processes

Config (defaults)
- DATABASE_URL=postgresql+psycopg2://postgres:postgres@localhost:5432/postgres
- REDIS_URL=redis://localhost:6379/0
- JWT_SECRET=supersecret; JWT_EXPIRE_MIN=30; JWT_REFRESH_EXPIRE_DAYS=14
- CORS_ORIGINS=http://localhost:8080,http://localhost:5173
- BINANCE_MARKET=spot; CCXT_RATE_LIMIT_MS=500

How to run (non-docker)
- ./scripts/setup_local.sh
- ./scripts/start_local.sh

How to run (docker)
- cd docker && docker compose up --build
